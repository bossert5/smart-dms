# Security Policy

Smart DMS handles documents, metadata, OCR output, email content and AI
provider secrets. Please do not publish vulnerability details in public
issues before the maintainer has had time to respond.

## Reporting a Vulnerability

Report security issues privately to the repository maintainer. If GitHub
private vulnerability reporting is enabled for the repository, prefer that
channel. Otherwise contact the maintainer through the profile linked from the
GitHub repository.

Please include:

- Affected version or commit.
- A clear description of the issue.
- Reproduction steps or proof of concept, if available.
- Expected impact.

## Supported Versions

Security fixes are expected to target the current main branch unless a
maintained release branch is announced.

## Sensitive Data

Never include real documents, OCR output, API keys, JWTs, cookies, passwords,
email content or private infrastructure details in public reports.
